## 📱 手机访问说明

### 问题：手机打开网址后没有自动创建账户

**原因**：
1. 手机访问会自动跳转到 `mobile.html`（移动端页面）
2. 移动端页面是用户前端，不是管理后台
3. **账户创建是在服务器启动时自动完成的**，不是访问网站时创建

**解决方案**：

### 方案1：在手机上直接访问后台（推荐）

在手机浏览器中直接输入：

```
https://aipigeonai.zeabur.app/admin.html
```

这样就能访问管理后台，使用默认账户登录：
- 用户名：`admin`
- 密码：`admin123`

### 方案2：检查服务器是否正常启动

账户创建是在服务器启动时自动完成的。如果账户没有创建，可能是：

1. **服务器没有正常启动**
   - 在Zeabur控制台查看 "Logs"（日志）标签页
   - 查找是否有以下信息：
     ```
     ✅ 默认管理员账户已创建
       用户名: admin
       密码: admin123
     ```
   - 或者：
     ```
     默认管理员账户已存在
     ```

2. **如果日志中没有账户创建信息**
   - 检查服务器是否正常启动
   - 查看是否有错误信息
   - 可能需要重新部署

### 方案3：手动创建账户（如果自动创建失败）

如果自动创建失败，可以通过以下方式手动创建：

1. **访问注册页面**（如果存在）
   - 访问：`https://aipigeonai.zeabur.app/`（PC端）
   - 查找注册功能

2. **或者检查数据库**
   - 如果使用Supabase，可以在Supabase控制台查看users表
   - 确认是否有admin账户

## ⚠️ 常见问题

### Q1: 手机访问后跳转到mobile.html，找不到登录入口

**A**: 这是正常的。移动端会自动跳转到 `mobile.html`（用户前端）。要访问管理后台，需要：

1. 在手机浏览器中直接输入：`https://aipigeonai.zeabur.app/admin.html`
2. 或者使用PC浏览器访问后台

### Q2: 登录时提示账户不存在

**A**: 可能的原因：

1. **服务器未正常启动**
   - 检查Zeabur日志，确认服务器是否启动
   - 查看是否有账户创建的信息

2. **账户创建失败**
   - 检查数据库连接是否正常
   - 查看日志中的错误信息

3. **解决方案**
   - 重新部署服务
   - 或者手动在数据库中创建账户

### Q3: 如何确认账户是否已创建？

**A**: 

1. **查看Zeabur日志**：查找 "默认管理员账户已创建" 或 "默认管理员账户已存在"
2. **尝试登录**：使用 `admin` / `admin123` 尝试登录
3. **检查数据库**：如果使用Supabase，在Supabase控制台查看users表

## 📞 需要帮助？

如果仍然无法创建账户或登录：

   - 查看是否有错误信息
