## 📊 常用操作

### 查看异常用户

```bash
GET /api/enhanced/users/anomalies
```

返回：
- 高风险用户数量
- 中等风险用户数量
- 低风险用户数量
- 详细异常列表

### 检查数据质量

```bash
GET /api/enhanced/data/quality
```

返回：
- 数据质量评分
- 各类型数据完整度
- 数据问题列表
- 优化建议

### 监控系统性能

```bash
GET /api/enhanced/performance/monitor
```

返回：
- 内存使用情况
- CPU使用情况
- API响应时间
- 性能问题列表

### 检测安全威胁

```bash
GET /api/enhanced/security/threats
```

返回：
- 威胁总数
- 高/中/低风险威胁数量
- 详细威胁列表
- 防护建议

### 生成综合报告

```bash
GET /api/enhanced/reports/comprehensive?period=daily
```

返回：
- 系统健康报告
- 用户管理报告
- 数据质量报告
- 性能报告
- 安全报告
- AI生成的总结和建议

## ⚙️ 配置说明

### 自动启用

所有增强功能在系统启动时自动启用，无需配置。

### 定时任务

定时任务已自动启动，包括：

| 任务 | 时间 | 说明 |
|------|------|------|
| 用户异常检测 | 每30分钟 | 自动检测异常用户 |
| 数据质量检查 | 每小时 | 检查数据质量 |
| 性能监控 | 每15分钟 | 监控系统性能 |
| 安全威胁检测 | 每10分钟 | 检测安全威胁 |
| 数据清理 | 每天3点 | 清理过期数据 |
| 数据备份 | 每天2点 | 自动备份 |
| 生成报告 | 每天1点 | 生成综合报告 |

### 自定义配置

如需修改配置，编辑对应的模块文件：

- `user-manager.js` - 用户管理阈值
- `data-manager.js` - 数据保留策略
- `performance-optimizer.js` - 性能阈值
- `security-monitor.js` - 安全规则
