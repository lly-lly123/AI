# 🔧 修复部署错误 - 第3步：配置构建命令

## ⚠️ 问题原因

错误：`Cannot find module 'helmet'`

**原因**：Zeabur的构建命令可能没有正确执行，导致依赖没有安装。

---

## ✅ 解决方案：在Zeabur中手动配置构建命令

### 操作步骤：

1. **返回服务页面**：
   - 点击左侧的 **"< Back"** 或 **"ai"** 服务名称
   - 返回到服务的 Overview 页面

2. **打开Settings（设置）**：
   - 点击页面顶部的 **"Settings"** 标签页

3. **配置Build Command**：
   - 找到 **"Build Command"** 或 **"Build Settings"** 部分
   - 在输入框中填写：
     ```
     cd backend && npm install
     ```

4. **配置Start Command**：
   - 找到 **"Start Command"** 或 **"Run Command"** 部分
   - 在输入框中填写：
     ```
     cd backend && npm start
     ```

5. **配置Root Directory（如果需要）**：
   - 找到 **"Root Directory"** 或 **"Working Directory"**
   - 设置为：`.` （点号，表示项目根目录）

6. **保存配置**：
   - 点击 **"Save"** 或 **"Update"** 按钮
   - Zeabur会自动重新部署

---

## 📋 配置检查清单

请确认以下配置：

- [ ] Build Command: `cd backend && npm install`
- [ ] Start Command: `cd backend && npm start`
- [ ] Root Directory: `.` （项目根目录）
- [ ] Node Version: `18.x` 或 `20.x`（如果可以选择）
