---

## 🚀 方案A：如果你已经有Token（推荐）

### 立即操作：

1. **在终端中，点击密码输入框**（光标所在位置）

2. **粘贴你的Token**：
   - 按 `Command + V` 粘贴
   - 或者右键点击 → "粘贴"
   - Token应该以 `ghp_` 开头

3. **重要提示**：
   - ⚠️ 终端输入密码时**不会显示任何字符**（这是正常的，为了安全）
   - 你粘贴后看不到任何内容，这是对的
   - 直接按**回车键**即可

4. **如果粘贴成功**，终端会继续执行，开始推送代码

---

## 🔑 方案B：如果你还没有Token（需要创建）

### 快速步骤（约2分钟）：

#### 第1步：打开浏览器创建Token

1. **打开新标签页**，访问：
   ```
   https://github.com/settings/tokens
   ```

2. **点击按钮**：
   - 点击绿色的 "Generate new token"
   - 选择 "Generate new token (classic)"

3. **填写信息**：
   - **Note**: 输入 `PigeonAI推送`
   - **Expiration**: 选择 `90 days` 或 `No expiration`
   - **权限**: ✅ 勾选 **`repo`**（Full control of private repositories）

4. **滚动到底部**，点击绿色的 "Generate token"

5. **立即复制Token**：
   - 会显示一个以 `ghp_` 开头的长字符串
   - 点击复制按钮，或选中后 `Command + C` 复制
   - ⚠️ **只显示一次！**立即复制保存

#### 第2步：回到终端，粘贴Token

1. **切换回终端窗口**（Command + Tab）

2. **在密码提示处**（光标所在位置）：
   - 按 `Command + V` 粘贴Token
   - 或右键点击 → "粘贴"

3. **按回车键**

4. **开始推送**：终端会显示推送进度

## ❌ 如果提示"认证失败"

### 可能的原因：

1. **使用了账户密码**（应该用Token）
   - ✅ 解决方法：使用Token，不是密码 `wikhI0-vymvyr-viptef`

2. **Token输入错误**
   - ✅ 解决方法：重新创建Token并复制粘贴

3. **Token过期**
   - ✅ 解决方法：创建新Token

4. **Token没有repo权限**
   - ✅ 解决方法：确保创建Token时勾选了`repo`权限

### 重新尝试：

如果认证失败，终端会回到命令提示符。重新执行：
```bash
git push -u origin main
```
然后输入Token。
