# ⏳ 部署构建中 - 等待时检查清单

## 📊 当前状态

- ✅ **构建正在进行中**（BUILDING）
- ⏳ **等待构建完成**（通常1-3分钟）
- 🔍 **构建完成后需要查看Runtime Logs**

---

## ✅ 在等待构建时，先检查这些配置

### 1. 检查环境变量（最重要！）

**操作步骤：**
1. 在Zeabur控制台，点击 **"Variable"** 标签页（顶部导航栏）
2. 检查是否已配置以下环境变量（至少一个）：

   ```
   ZHIPU_API_KEY_EVO=你的智谱API密钥
   ```
   或
   ```
   ZHIPU_API_KEY_ADMIN=你的智谱API密钥
   ```
   或
   ```
   ZHIPU_API_KEY=你的智谱API密钥
   ```

3. **如果未配置**：
   - 点击 **"Add Variable"** 或 **"添加变量"**
   - 变量名：`ZHIPU_API_KEY_EVO`（或 `ZHIPU_API_KEY_ADMIN`）
   - 变量值：你的智谱API密钥
   - 点击 **"Save"** 保存

**⚠️ 注意：** 大多数崩溃都是因为缺少这个环境变量！

---

### 2. 检查启动命令

**操作步骤：**
1. 点击 **"Settings"** 标签页
2. 找到 **"Start Command"** 或 **"启动命令"** 字段
3. **正确配置应该是：**
   ```
   cd backend && npm start
   ```
   或
   ```
   npm start
   ```
4. **确保没有中文输入法错误**（如 `cd后端&&npm启动`）
5. 如果错误，修改后点击 **"Save"**

---

### 3. 检查根目录设置

**操作步骤：**
1. 在 **"Settings"** 标签页
2. 找到 **"Root Directory"** 或 **"根目录"** 字段
3. **应该是：** `.` 或留空（表示项目根目录）
4. 如果错误，修改后保存

---

## 🔍 构建完成后的操作

### 步骤1：查看构建结果

等待构建完成后，状态会变为：
- ✅ **"Running"**（运行中）- 部署成功！
- ❌ **"Crashed"** 或 **"Crashed Retrying"** - 需要查看错误日志

---

### 步骤2：如果状态是 "Crashed"

1. **点击 "Runtime Logs" 标签页**（在 "Build Logs" 旁边）
2. **向下滚动**查看完整日志
3. **查找红色错误信息**，特别是：
   - 最后几行的错误
   - `Error:` 开头的行
   - `Cannot find module` 错误
   - `undefined` 相关错误
   - 端口相关错误

4. **截图错误日志发给我**，我会告诉您如何修复

---

### 步骤3：如果状态是 "Running"

🎉 **恭喜！部署成功！**

验证步骤：
1. 访问你的域名：`aipigeonai.zeabur.app`
2. 检查页面是否正常加载
3. 测试功能是否正常
